## Générez votre propre clé API

La première chose à faire pour accéder à l'API d'Ollabot est de générer votre propre clé API. Vous pouvez générer votre clé API en suivant les étapes ici : [Générer une clé API](https://app.ollabot.com/docs/api).

**Guide de l'API de Suppression**

L'API de Suppression de Chatbot vous permet de supprimer un chatbot existant en faisant une requête DELETE à l'endpoint `/api/v1/delete-chatbot`.

## Endpoint

**DELETE** https://app.ollabot.com/api/v1/delete-chatbot

## En-têtes de Requête

La requête API doit inclure les en-têtes suivants :

- **Authorization**: Bearer `<Your-Secret-Key>` - La clé secrète pour authentifier la requête API.
- **Content-Type**: application/json - Le type de contenu de la charge utile de la requête.

## Corps de la Requête

Le corps de la requête doit contenir le paramètre suivant :

- **botId** (string, obligatoire) : L'identifiant unique du chatbot à supprimer.

### Exemple de Requête

#### JavaScript

```javascript
const res = await fetch('https://app.ollabot.com/api/v1/delete-chatbot', {
  method: 'DELETE',
  headers: {
    Authorization: `Bearer <Your-Secret-Key>`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    botId: 'exampleId-123'
  })
});

const data = await res.json();

console.log(data); // { message: 'Chatbot deleted successfully' }
```

#### Python

```python
import requests

url = 'https://app.ollabot.com/api/v1/delete-chatbot'
headers = {
    'Authorization': 'Bearer <Your-Secret-Key>',
    'Content-Type': 'application/json'
}
data = {
    'botId': 'exampleId-123'
}

response = requests.delete(url, json=data, headers=headers)
print(response.json())
```

#### Shell

```bash
curl -X DELETE https://app.ollabot.com/api/v1/delete-chatbot \
  -H "Authorization: Bearer <Your-Secret-Key>" \
  -H "Content-Type: application/json" \
  -d '{"botId": "exampleId-123"}'
```

## Réponse

La réponse de l'API sera un objet JSON avec la structure suivante :

```json
{
  "message": "Chatbot

 deleted successfully"
}
```

## Gestion des Erreurs

En cas d'erreurs lors de la requête API, des codes de statut HTTP appropriés seront retournés avec des messages d'erreur dans le corps de la réponse. Assurez-vous de gérer ces erreurs de manière appropriée dans votre application.
